Analyzing: POST /orders

Operation: PostOrders
Summary: Create a new order, pickup request or rate quote
Description: Creates one or more order, pickup request or rate quote records.

The following fields will be defaulted based on the following app config options:

* CSERV.EXE > Default Bill To

* CSERV.EXE > Default Site

 Other configuration options can affect how certain field values are populated if not provided in the request:

* startZone - defaults to the provided shipper zone or the shipper's default zone

* endZone - defaults to the provided consignee zone

* serviceLevel - defaults to Codes Maintenance > Site Setup > Service Level default

The billToCode can only be updated using the billTo override (R) option.

The interliner (I) option for billTo can only be used if there is at least one interliner record with that has a receivable chargesType option set.

Duplicate PO is not allowed on traceNumber for the same billTo when the app config option CSERV.EXE > Check PO is set.

 Extra Stops (XStop) child bills can be created using the masterOrderId query parameter in TruckMate versions 25.1.2 and higher.

* The order(s) included in the request body will be created as xStops for the orderId that is used as the masterOrderId query parameter value.

* The masterOrderId must be a valid posted orderId that is not a pickup request or quote and cannot be interfaced to billing.

* The orders included in the request body when creating xStops must have a billToCode, siteId and currencyCode that matches the masterOrderId.

 Orders and rate quotes can be viewed in the  Customer Service application.

Pickup requests can be viewed in the Order Preprocessor application.

App config options can be viewed in the Application Configurator application.
