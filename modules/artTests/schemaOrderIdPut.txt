Analyzing: PUT /orders/{orderId}

Operation: PutOrder
Summary: Update an order, pickup request or rate quote
Description: Updates one or more properties of the specified order, pickup request or rate quote record.

Only active orders, or pickup requests associated with the authentication key used to access the endpoint can be updated.

* Approved pickup requests cannot be updated.

* Orders must be posted (have a bill number) and cannot be cancelled or interfaced to billing.

Not all of the Security Configuration > Security > Business Events settings for Customer Service which allow or prevent editing of orders based on tab, status and other options are used by the APIs.

The requested body may include child resources such as details, dangerousGoods, customDefs, etc. Keep in mind when a resource is specified in the body, it will do a complete replace of that resource. In other words, it will DELETE ALL records of the specified resource in the order, and insert new ones from the request body.

To update an individual child record use the PUT method directly against the child resource record itself.

* For example, to update an individual detail line use PUT /orders/{orderId}/details/{orderDetailId}

Note that the API cannot edit or delete a Cash-On-Delivery (COD) record for an order but it can update the codAmount. If no COD record exists for the order, the API will create it.

The estimatedDeliveryDate will be updated based on the following app config option:

* CSERVE.EXE > Reset Estimated Delivery Date Value When Delivery Date of Freight Bill Changed

The billToCode can only be updated using the billTo override (R) option.

The interliner (I) option for billTo can only be used if there is at least one interliner record with that has a receivable chargesType option set.

Duplicate PO is not allowed on traceNumber for the same billTo when the app config option CSERV.EXE > Check PO is set.

Orders and rate quotes can be viewed in the Customer Service application.

Pickup requests can be viewed in the Order Preprocessor application.

COD records can be viewed in the Customer Service application > COD tab.
